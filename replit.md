# Entwine Store

A full-stack e-commerce clone of [entwinestore.com](https://www.entwinestore.com/) — handcrafted Indian block-print clothing. Built as a MERN stack app (MongoDB, Express, React, Node.js).

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (starts local MongoDB + serves on port 8080)
- `pnpm --filter @workspace/entwine-store run dev` — run the React frontend (Vite)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- **Frontend**: React 19 + Vite 7, Tailwind CSS v4, Wouter routing, TanStack Query v5
- **API**: Express 5 + OpenAPI spec (contract-first), Orval codegen
- **DB**: MongoDB (local, `mongodb://localhost:27017/entwine`) + Mongoose
- Validation: Zod (`zod/v4`)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI contract (source of truth for all endpoints)
- `lib/api-client-react/src/generated/` — auto-generated React Query hooks
- `artifacts/api-server/src/` — Express routes, Mongoose models, seed data
  - `models/` — Product, Category, Cart models
  - `routes/` — products, categories, cart, store
  - `seed.ts` — 6 categories + 16 products seed
  - `lib/db.ts` — MongoDB connection
- `artifacts/entwine-store/src/` — React frontend
  - `pages/` — HomePage, ShopPage, CategoryPage, ProductDetailPage, CartPage, SalePage, AboutPage, ContactPage
  - `components/` — Navbar, Footer, ProductCard
  - `hooks/use-cart-session.ts` — localStorage-based cart session management

## Architecture decisions

- **Contract-first API**: OpenAPI spec lives in `lib/api-spec`, all client hooks are generated by Orval — never hand-write fetch calls.
- **Session-based cart**: Cart uses a UUID stored in localStorage (`entwine_session_id`) — no auth required.
- **MongoDB embedded startup**: The API dev script starts `mongod` automatically via `--fork` before launching the server.
- **Path-based routing**: Global reverse proxy routes `/api` → Express, `/` → Vite. No hardcoded ports in application code.

## User preferences

- Brand: dark red primary `#77201D`, warm cream background, Jost (headings) + Inter (body)
- INR currency formatting
- Indian e-commerce style (block-print / ethnic fashion)

## Gotchas

- MongoDB must be running before the API server starts — the `dev` script handles this automatically.
- Always run `pnpm --filter @workspace/api-spec run codegen` after changing `openapi.yaml`.
- Do not use `pnpm dev` at the workspace root — individual artifact workflows handle env vars.
